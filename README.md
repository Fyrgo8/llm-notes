# LLM Notes

这个仓库现在不是单一形态的“笔记目录”，而是几类不同用途的 LLM 学习资产放在同一个仓库里：

- 面向面试与知识点复习的结构化八股
- 面向具体项目与论文的长篇学习笔记
- 面向长期积累的 Obsidian 论文阅读 vault
- 面向方向判断的方法论笔记

如果只看当前 GitHub 首页，很容易误以为仓库主体只有几篇根目录 Markdown。实际上，仓库里最重的内容在 `面试核心高频问题/`、`大模型算法八股/` 和 `论文阅读笔记/` 这几块。

## 仓库结构

### `面试核心高频问题/`

这一组内容更像“高频题精炼版”。它不是把所有材料原样堆上来，而是按真实面试高频主题重新组织，适合快速回顾主线。

当前包含 10 个主题：

- Transformer 与 Attention
- 位置编码与长度外推
- 强化学习与对齐
- 参数高效微调
- 分布式训练
- 推理优化
- 模型架构与设计选择
- Agent 与 RAG
- 数据工程
- 评估体系

如果你的目标是面试准备，建议先从这一组开始，而不是一上来直接进 `大模型算法八股/`。

### `大模型算法八股/`

这是仓库里体量最大的一块，偏“系统知识库”而不是“速记版问答”。目录下不只是几篇总览文档，而是完整分层内容，覆盖：

- `01.大语言模型基础`
- `02.大语言模型架构`
- `03.训练数据集`
- `04.分布式训练`
- `05.有监督微调`
- `06.推理`
- `07.强化学习`
- `08.检索增强RAG`
- `09.大语言模型评估`
- `10.大语言模型应用`
- `98.相关课程`
- `99.参考资料`

这一部分更适合两类场景：

- 你想系统补全某一块知识，不满足于“面试答案级别”的概括
- 你已经知道自己要看什么主题，准备沿着一个主题往深处钻

它内部还带有大量图片、配套 PDF 和按主题拆开的子文档，所以它更像一套可持续扩展的本地知识库。

### `大模型算法方向选择/`

这一块不是知识点讲解，而是“往哪个方向投入时间”的方法论笔记。当前主要包括：

- `AI_Infra_2026深度版.md`
- `AI可入方向.md`
- `大模型应用算法落地方法论.md`

如果你关心的不只是“学什么”，而是“下一阶段该把时间压在哪条路线上”，这一组更值得优先看。

### 根目录长篇笔记

根目录有几篇更偏项目 / 论文 / 工程实践的长篇笔记：

- [MiniMind_笔记.md](./MiniMind_%E7%AC%94%E8%AE%B0.md)
  围绕 MiniMind 的训练到推理全链路，适合建立“小模型从零到可跑”的整体认知。
- [MiniResearcher_学习笔记.md](./MiniResearcher_%E5%AD%A6%E4%B9%A0%E7%AC%94%E8%AE%B0.md)
  聚焦低资源多轮搜索 Agent 的 RL 训练实践，内容更偏实验、系统实现和问题拆解。
- [agent-harness-engineering-survey-cn.md](./agent-harness-engineering-survey-cn.md)
  《Agent Harness Engineering: A Survey》中文全文翻译，偏 Agent 基础设施与执行框架视角。

这几篇和 `面试核心高频问题/`、`大模型算法八股/` 的关系不是互相替代，而是承担不同任务：

- 八股负责铺知识面
- 长篇笔记负责把某条完整实践路径讲透

## `论文阅读笔记/`：一套独立的论文阅读 Vault

`论文阅读笔记/` 不是把论文摘要单独放一个文件夹这么简单，而是一套独立的 Obsidian 工作流。它的重点不只是“存论文笔记”，而是把阅读、索引、摘录、主题沉淀和模板组织成一个长期可维护系统。

核心目录如下：

- `00 Inbox`
  暂存未整理的论文、想法和待处理材料。
- `01 MOC`
  整个 vault 的导航层，包含论文总览、论文看板、最近论文、研究主题、作者索引、阅读状态看板等入口。
- `02 Papers`
  单篇论文笔记与精读结果，当前已有 `EnvScaler`、`ET-Agent`、`Tool-Star` 等条目。
- `03 Topics`
  按主题聚合的跨论文笔记，例如 `Agentic RL`、`Tool Use`、`GRPO`、`RLHF`。
- `04 Authors`
  作者维度归档。
- `05 Venues`
  会议 / 期刊维度归档。
- `06 Daily Notes`
  阅读过程中的日常记录。
- `07 Templates`
  论文笔记、深读笔记、主题笔记等模板。
- `08 Attachments`
  附件与辅助材料。
- `09 Excerpts`
  摘录、intake 和记要。
- `copilot`
  配套 prompt 资产。
- `.obsidian`
  Vault 配置与插件配置。

如果你的目标是把论文阅读积累成长期资产，而不是只看完一篇就结束，这一块才是仓库里最值得持续投入的部分。

## 推荐阅读路径

不同目标下，入口不一样。

1. 如果你在准备面试：
   先看 `面试核心高频问题/`，再按需要回到 `大模型算法八股/` 对应章节补深度。
2. 如果你想系统补 LLM 知识：
   直接进 `大模型算法八股/`，按主题从基础、架构、训练、推理一路读下去。
3. 如果你更关心项目实践：
   从 [MiniMind_笔记.md](./MiniMind_%E7%AC%94%E8%AE%B0.md) 和 [MiniResearcher_学习笔记.md](./MiniResearcher_%E5%AD%A6%E4%B9%A0%E7%AC%94%E8%AE%B0.md) 开始。
4. 如果你要进入论文阅读工作流：
   先看 `论文阅读笔记/01 MOC/论文总览.md`，再看 `论文阅读笔记/01 MOC/论文看板.md`，最后进入 `论文阅读笔记/02 Papers/` 的单篇笔记。
5. 如果你要判断下一阶段方向：
   直接看 `大模型算法方向选择/`。

## 使用方式

- 只想读内容：直接在 GitHub 浏览 Markdown 即可。
- 想完整使用论文库：用 Obsidian 打开 `论文阅读笔记/`。
- 想迁移这套结构到自己的知识库：优先参考 `论文阅读笔记/01 MOC/` 的导航组织方式和 `论文阅读笔记/07 Templates/` 的模板设计。
