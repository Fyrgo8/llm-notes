# Agent 与 RAG

> 优先级：P2（加分项）
> 覆盖范围：Agent 架构、工具调用、规划与记忆、RAG 全流程、Embedding 模型、CoT
> 关联背景：美团 Agent 开发实习

### 1. Agent 的四要素架构

LLM Agent 的标准框架由四个模块组成：LLM 作为大脑负责推理决策，Tools 提供与外部世界交互的能力（搜索、计算、API 调用），Memory 维护短期上下文和长期经验，Planning 负责任务分解和执行策略。

ReAct（Reasoning + Acting）是最经典的 Agent 范式：模型交替输出 Thought（当前推理）→ Action（工具调用）→ Observation（工具返回），循环直到任务完成。这种结构的好处是推理过程完全可追溯，便于调试和人工审核。

### 2. Function Calling 的实现机制

工具调用的技术实现通常是：将可用工具的定义（JSON Schema 格式的函数签名、参数描述）拼接到 system prompt 中，模型在需要调用工具时输出特定格式的结构化文本（函数名 + 参数 JSON），框架解析后执行工具并将结果返回给模型继续生成。

训练方式是在 SFT 阶段混入大量工具调用格式的数据——模型需要学会两件事：何时应该调用工具（而非直接回答），以及如何正确填写参数。Qwen2.5 和 GPT-4 的 function calling 能力均通过这种方式训练得到。

### 3. 规划策略

Chain-of-Thought 是最基础的规划：让模型"展示思考过程"，将复杂问题分解为顺序子步骤。Zero-shot CoT 只需在 prompt 末尾加"Let's think step by step"就能激活这一能力——本质是利用自回归的序列依赖性，前面的推理步骤为后续生成提供了上下文线索。

更复杂的规划策略包括 Tree-of-Thought（分支搜索 + 回溯）、Plan-and-Execute（先生成完整计划再逐步执行）、Reflexion（执行后反思、修正下一次行动）。在实际 Agent 系统中，选择哪种策略取决于任务的确定性——确定性高的任务（如数据查询）用线性 CoT 即可，开放性探索任务（如研究助手）需要更灵活的搜索策略。

### 4. RAG 全流程

RAG 解决 LLM 的三个根本问题：知识截止（训练数据有时间界限）、幻觉（生成不存在的事实）、私有数据（无法通过公开训练获取）。

标准流程分离线和在线两部分。离线阶段将文档分块（chunk）、通过 embedding 模型编码为向量、存入向量数据库。在线阶段将用户 query 编码为向量，检索 Top-K 相关 chunk，拼接到 prompt 中作为上下文，LLM 基于检索到的信息生成回答。

### 5. RAG 各环节的优化

**分块**是 RAG 质量的基础。固定长度切分简单但可能切断语义完整的段落；递归字符切分按段落→句子→字符逐级降级；语义切分用 embedding 相似度检测主题转换断点——相邻段的 cosine 相似度骤降处即为分块边界。

**检索**环节，单一的稠密向量检索在关键词精确匹配场景下不如传统 BM25。混合检索（稠密 + 稀疏）取两者之长。Reranker（交叉编码器）对初检的 Top-K 做精排——cross-encoder 将 query 和 passage 拼接后联合编码，精度远高于 bi-encoder 的独立编码，但计算量限制了只能用于重排少量候选。

**生成**环节，"Lost in the Middle" 现象表明模型对长上下文中间位置的信息利用率低，因此应将最相关的 chunk 放在上下文的开头和结尾。上下文压缩（过滤明显无关段落）可以减少噪声对生成的干扰。

### 6. Embedding 模型

检索质量的上界由 embedding 模型决定。BGE 系列是中文场景最强的开源选择（bge-large 1024 维），E5 在多语言场景表现优异。训练方式是对比学习（InfoNCE loss）：正例对（query 和相关 passage）拉近、负例对推远。

关键工程细节：embedding 模型的训练数据分布必须与实际检索场景匹配——用通用语料训练的 embedding 在垂域场景效果可能大打折扣，因此需要用领域数据做 fine-tune。

### 7. 幻觉的分类与成因

幻觉按严重程度可分为两大类：**事实性幻觉**（Factual Hallucination）——模型编造与现实不符的"事实"（如杜撰不存在的论文、捏造人物经历），以及**忠实性幻觉**（Faithfulness Hallucination）——模型的回答与给定上下文/文档不一致（如摘要中包含原文没有的信息）。

成因来自多个层面。数据层面：预训练语料中本身包含虚假信息、过时信息、相互矛盾的描述。模型层面：自回归生成的"左到右"机制导致模型无法回溯修正前文错误；解码时的随机采样可能在高不确定区域采到低概率但看似合理的续写。训练层面：交叉熵 loss 鼓励模型对每个位置都给出高置信度预测，而不是"表达不确定性"。

**检测手段**包括：基于知识库的事实核查（将生成文本分解为原子命题逐一验证）、Self-consistency 检验（多次采样后对比一致性，不一致的部分大概率为幻觉）、以及 NLI 模型判断生成内容是否被源文档蕴含。

### 8. Hallucination 的系统性缓解

缓解策略需要在训练和推理多个环节协同作用：

训练阶段，RLHF 中标注者对含幻觉的输出给低分，模型学会在不确定时表达"我不确定"而非强行编造。RLHF 训练的 reward model 本身也需要在事实性维度上有足够的区分能力。

推理阶段的策略更加多样：RAG 提供外部事实依据，将生成的"依据来源"从模型参数转移到检索文档，显著降低编造概率；Self-consistency（多次采样取多数投票）过滤掉低置信度的随机错误；工具调用允许模型在需要事实验证时主动查询外部知识源；Constrained decoding 在需要结构化输出时保证格式可靠。

在实际 Agent 系统中，最有效的防幻觉策略是"让模型不要直接回答超出其确定性的问题，而是调用工具获取事实后再综合回答"。这也是 ReAct 范式优于纯 CoT 的关键原因——每一步都有 Observation 作为外部锚点，限制了推理链"漂移"的空间。

### 9. RAG vs 微调：何时选择哪条路

面试常见问题是"什么情况下该用 RAG、什么情况下该微调"。本质区别在于知识的来源和更新频率。

RAG 适合的场景：知识更新频繁（如新闻、文档库持续增长）、需要精确引用来源（法律、医疗合规）、私有数据不希望写入模型参数（安全性考虑）、以及知识库规模远大于模型容量的情况。RAG 的上限取决于检索质量——如果检索召回不到正确文档，再强的 LLM 也无法补救。

微调适合的场景：需要模型学会新的行为模式或输出格式（如 JSON 结构化输出、特定风格的文案）、任务涉及的"知识"是隐性的推理能力而非显式事实、或延迟要求极高（RAG 引入额外的检索延迟）。LoRA 微调对于"教模型新技能"效率很高，但不适合注入大量事实性知识——几千条训练数据无法可靠地改变模型的世界知识。

实践中二者常常联用：微调让模型学会"如何利用检索结果回答问题"（如 RAFT 方法），RAG 提供实时知识。

### 10. CoT 推理的深入分析

Chain-of-Thought 的本质是利用自回归模型的序列依赖性——将中间推理步骤显式化为 token 序列，使得后续 token 的生成能"看到"前面的推理结果，从而突破单步 forward pass 的计算深度限制。

**Zero-shot CoT** 只需 prompt 末尾加"Let's think step by step"即可激活。原理是预训练语料中"解题过程"类文本建立了"先分析再给答案"的分布模式，这句 prompt 将采样导向该模式。Wei et al. (2022) 发现这种能力仅在 100B+ 参数模型上才稳定出现（涌现能力的一个实例）。

**Few-shot CoT** 在 prompt 中给出带推理步骤的示例，是更可靠的激活方式。示例中推理步骤的格式和粒度直接影响模型输出质量——过粗会跳步导致错误，过细则不经济。

**Self-Consistency** 是 CoT 的增强：对同一问题多次采样（用不同 temperature），对最终答案投票取多数。本质是将采样的随机性从"噪声源"转化为"集成学习"的多样性。在 GSM8K 等数学任务上，self-consistency 比单次 CoT 提升 10-15%。

**CoT 的局限**：计算开销线性增长（生成更多 token = 更多推理延迟）、对于不需要推理的事实查询反而可能引入不必要的"过度思考"错误、以及推理链本身可能包含看似合理但逻辑有误的步骤（"幻觉推理"）。
