# 分布式训练

> 优先级：P1（重要）
> 覆盖范围：数据并行、ZeRO、张量并行、流水线并行、3D 并行、混合精度
> 关联项目：MiniResearcher 使用 DeepSpeed ZeRO-2 + QLoRA，2×A100 80GB

### 1. 为什么需要分布式

以 7B 模型 fp16 为例：参数本身 14GB，Adam 优化器状态（fp32 参数副本 + 一阶/二阶动量）额外占 3 倍即 42GB，加上梯度 14GB，总计约 70GB。这还没有计算激活值——batch size 稍大就会 OOM。70B 模型更是单卡完全不可能。分布式训练的本质是把"放不下"或"算太慢"的问题分而治之。

### 2. 数据并行

最朴素的方案：每张卡持有完整模型副本，将 mini-batch 切分到各卡，各自前向/反向后通过 AllReduce 同步梯度。通信量为 $2 \times \text{model\_size}$（reduce-scatter + all-gather 各一次模型大小）。

数据并行的致命局限是：模型必须放进单卡。当模型超过单卡显存容量时，必须引入模型并行或 ZeRO。

### 3. ZeRO 三阶段

DeepSpeed 的 ZeRO（Zero Redundancy Optimizer）观察到数据并行中每张卡都冗余存储了完整的优化器状态/梯度/参数，本质上只需要各存 $1/N$。

ZeRO-1 只切分优化器状态：单卡显存从约 70GB（7B fp16）降到约 30GB。ZeRO-2 额外切分梯度，降到约 16GB。ZeRO-3 连参数也切分——每张卡只存 $1/N$ 的参数，需要时通过 AllGather 临时恢复，计算完即释放。代价是通信量增加（每步需要额外的 AllGather），但在高带宽互联下可接受。

PyTorch 原生的 FSDP（Fully Sharded Data Parallel）本质等价于 ZeRO-3，区别在于实现细节和 API 设计。DeepSpeed 配置更灵活，FSDP 与 PyTorch 生态集成更紧密。

### 4. 张量并行

将单层内部的权重矩阵切分到多卡。以 MLP 为例：第一个线性层 $Y = XW_1$ 按列切分 $W_1$（各卡独立计算部分列），第二个线性层 $Z = YW_2$ 按行切分 $W_2$（各卡持有部分行，计算后 AllReduce 求和）。Attention 层按 head 维度切分——不同卡负责不同 head 组，天然对齐 GQA 结构。

张量并行每层需要 2 次 AllReduce 通信，对带宽要求极高。实践中只在节点内使用（NVLink 带宽 600 GB/s），跨节点带宽不足以支撑。

### 5. 流水线并行

按层将模型切成多个 stage，不同卡负责不同 stage。数据以 micro-batch 流水方式通过各阶段。核心问题是 pipeline bubble——流水线启动和排空阶段有卡空闲。

GPipe 方案先做所有 micro-batch 的 forward 再做所有 backward，简单但峰值显存高（需缓存所有激活）。1F1B 方案交错执行 forward 和 backward，及时释放激活，显存更友好。Interleaved 1F1B 让每张卡负责多个非连续 stage（如第 1、5、9 层），进一步降低 bubble 比例。

Bubble 比例约为 $(p-1)/(m+p-1)$，其中 $p$ 为 stage 数，$m$ 为 micro-batch 数。增大 $m$ 可以摊薄 bubble，但受显存限制。

### 6. 3D 并行组合

千卡规模训练的标配方案是 TP + PP + DP 的正交组合：TP 在节点内（利用 NVLink 高带宽），PP 跨节点（每个 stage 间只需点对点通信），DP/ZeRO 在 TP×PP 分组之间做数据并行。

以 64 卡训练 70B 为例：TP=8（单节点 8 卡），PP=4（跨 4 个节点），DP=2（两组流水线各处理不同数据）。Megatron-LM 提供了高度优化的 TP/PP 实现，通常与 DeepSpeed 的 ZeRO 配合使用。

### 7. 混合精度训练

前向和反向用 bf16/fp16 计算（速度快、显存省），主权重和优化器状态用 fp32 存储（精度保障）。Loss scaling 在 fp16 场景下放大 loss 值防止梯度下溢——bf16 由于指数位更多（8 bit vs fp16 的 5 bit）不易溢出，现代 LLM 训练几乎全用 bf16。

### 8. 梯度累积与梯度检查点

**梯度累积**：将 $K$ 个 micro-batch 的梯度累加后做一次参数更新，等效于 $K$ 倍 batch size。用于单卡 batch 受显存限制时模拟大 batch 训练。

**梯度检查点（Activation Checkpointing）**：前向时只保存部分层的激活，反向时重计算中间层激活。用计算换显存，通常增加约 33% 计算时间但显存降低 60-70%。
