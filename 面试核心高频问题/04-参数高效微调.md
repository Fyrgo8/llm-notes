# 参数高效微调

> 优先级：P1（重要）
> 覆盖范围：LoRA 原理与工程细节、AdaLoRA、QLoRA、SFT 实践
> 关联项目：MiniResearcher 使用 QLoRA（4bit NF4）在 2×A100 上微调 Qwen2.5-3B

### 1. LoRA 的核心思想

全参数微调的权重更新 $\Delta W \in \mathbb{R}^{d \times d}$ 理论上是满秩矩阵，但 Aghajanyan et al. (2020) 发现预训练模型的"本征维度"（intrinsic dimension）很低——随机投影到几十维子空间后仍能有效学习。这说明微调实际只需要在一个低秩子空间中调整参数。

LoRA 将这一观察形式化：冻结预训练权重 $W_0$，用低秩分解 $\Delta W = BA$ 来近似全参数更新：

$$
h = W_0 x + \Delta W x = W_0 x + BAx
$$

其中 $A \in \mathbb{R}^{r \times d}$ 用高斯初始化，$B \in \mathbb{R}^{d \times r}$ 零初始化。$B$ 零初始化保证训练开始时 $\Delta W = 0$，模型行为与预训练完全一致，后续渐进式学习增量。参数量从 $d^2$ 降到 $2dr$，当 $r=8, d=4096$ 时仅为全参数的 0.39%。

### 2. LoRA 应用在哪些层

原论文的消融实验表明同时在 $W_q$ 和 $W_v$ 上加 LoRA 效果最佳。后续实践（如 LLaMA-Factory 的默认配置）发现对全部 Q/K/V/O 和 MLP 层都加 LoRA 效果更稳健——尤其在复杂任务上，MLP 层的适配贡献不可忽略。

另一个经验结论：增加 LoRA 的覆盖层数比增大单层的秩 $r$ 更有效。$r=4 \sim 16$ 在多数任务上已足够，盲目增大 $r$ 不一定能覆盖更有意义的子空间。

### 3. LoRA 的推理与多任务切换

训练完成后将 $BA$ 合并到原权重：$W_{merged} = W_0 + BA$，推理时无任何额外延迟或显存开销——这是 LoRA 相比 Adapter（串行插入额外模块）的关键优势。

多任务场景下可以不合并，保留多个 adapter 权重按需加载（LoRA switching）。vLLM 已原生支持多 LoRA adapter 并发服务，通过 CUDA kernel 实现高效的 batched LoRA 计算。

### 4. AdaLoRA：自适应秩分配

LoRA 对所有层使用相同的秩 $r$，但不同层/不同矩阵对下游任务的重要性差异显著。AdaLoRA 将增量更新参数化为 SVD 形式 $\Delta W = P \Lambda Q$，训练过程中根据重要性得分动态裁剪奇异值——重要的层保留高秩，不重要的层裁到更低秩。

为避免对大矩阵做精确 SVD 的计算开销，AdaLoRA 在训练损失中加入正交性惩罚项来维持 $P$、$Q$ 的正交结构，从而以近似方式稳定训练。实验表明在参数预算为 0.3M 时，AdaLoRA 在 RTE 数据集上比固定秩 LoRA 高 1.8%。

### 5. QLoRA：4bit 量化 + LoRA

QLoRA 在 LoRA 基础上引入三个技术使大模型微调的显存门槛大幅降低：

**4-bit NormalFloat（NF4）量化**。权重分布近似正态，NF4 是信息论意义上对正态分布最优的 4bit 数据类型——将量化 bin 按正态分布的分位点划分，使每个 bin 内的概率质量相等。实证优于 4bit int 和 4bit float。

**双量化（Double Quantization）**。第一次量化产生的缩放常数（每 64 个参数一个 fp32 常数）本身再做一次 8bit 量化，将额外开销从 0.5 bit/param 降到 0.127 bit/param。

**分页优化器（Paged Optimizers）**。利用 NVIDIA 统一内存的 CPU-GPU 分页机制，在显存不足时自动将优化器状态卸载到 CPU，需要时再加载回来。

效果：LLaMA-65B 的微调从需要 780GB 降到单张 48GB GPU 可行；7B 模型在 24GB 显卡上即可训练。QLoRA 的精度损失可忽略——实验证明 NF4 + LoRA 微调后的效果与 fp16 全参微调持平。

### 6. PEFT 方法体系全景

参数高效微调并非只有 LoRA 一条路线。按照"额外参数插在哪里"可以划分为三大类：

**Adapter 类方法**。在 Transformer 每层的 Attention 和 FFN 之后插入小型全连接瓶颈模块（down-projection → 非线性 → up-projection）。Houlsby Adapter 在每个子层后都插入，Pfeiffer Adapter 只在 FFN 后插一个。推理时无法合并（模块串行在主路径上），因此会引入约 5-10% 的推理延迟。参数量约为原模型的 0.5-2%。

**Prefix/Prompt Tuning 类方法**。在输入序列前拼接可学习的"虚拟 token"，冻结全部模型参数只训练这些前缀。Prefix Tuning（Li & Liang, 2021）在每层 attention 的 K/V 前面都拼接前缀向量（通常 10-200 个），相当于提供了每层独立的"soft 提示信息"。Prompt Tuning（Lester et al., 2021）更简化——只在输入 embedding 层前拼虚拟 token，不深入每一层。P-Tuning v2 验证了"在每层都加 prefix"的做法在各种规模模型上都能逼近全参微调效果。

**对比总结**。LoRA 推理零延迟且支持多任务快速切换，适合部署场景。Adapter 实现简单但推理有额外开销，适合离线批量推理。Prefix/Prompt Tuning 参数最少（几千到几十万），适合资源极度受限或需要"一模多租户"的场景——不同任务只需切换一组前缀向量，base model 完全共享。

工程实践中 LoRA 已是默认选择。Adapter 和 Prefix Tuning 更多出现在面试"你知道哪些 PEFT 方法"的横向对比中——理解每种方法"参数加在哪、推理时能否去掉、适合什么规模"就够用了。

### 7. SFT 阶段的关键工程实践

**数据质量压倒数量**。LIMA 用 1000 条精选数据、Alpaca 用 52K 条 GPT-4 生成数据，效果远好于百万级低质数据。

**Loss masking**。只在 response 部分计算 loss，prompt 部分不回传梯度。如果对 prompt 也算 loss，模型会"学会"如何生成 prompt 格式，而非学会如何回答。

**学习率与 Epoch**。通常比预训练低 1-2 个量级（2e-5 左右），训练 1-3 epoch。过多 epoch 导致灾难性遗忘——模型过拟合微调数据而丧失预训练的通用能力。

**Packing**。将多条短样本拼接到同一个 sequence 中（用特殊分隔符隔开），提高 GPU 利用率。需要配合 attention mask 确保不同样本间不互相 attend。

### 8. 灾难性遗忘与缓解

微调的本质矛盾在于：适配下游任务需要修改参数，但修改参数可能覆写预训练阶段积累的通用知识。缓解策略包括三个方向：参数冻结（LoRA/Adapter 只动小部分参数，大部分知识不受影响），数据混合（在 SFT 数据中混入少量预训练格式数据保持通用能力），以及正则化约束（如 EWC 对"重要"参数施加额外的 L2 惩罚阻止其偏移）。

实践经验表明，LoRA 的低秩约束天然提供了一种正则化效果——$\Delta W$ 的表达能力受限于秩 $r$，不太可能对预训练权重做剧烈改动，因此 LoRA 微调后的灾难性遗忘通常比全参微调轻微得多。
