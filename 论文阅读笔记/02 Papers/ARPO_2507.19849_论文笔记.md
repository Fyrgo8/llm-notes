# ARPO（arXiv:2507.19849）论文笔记

Guanting Dong et al. | arXiv v1, 2025 | [PDF](https://arxiv.org/pdf/2507.19849v1) | `Agentic RL` `Tool Use` `Adaptive Rollout` `Advantage Attribution`

## 问题与核心判断

ARPO 要解决的不是“模型不会调用工具”，而是**常规 trajectory-level RL 的采样与归因粒度不适合多轮工具交互**。GRPO、DAPO 一类方法通常从同一问题独立采样若干条完整轨迹，再用最终答案奖励比较整条轨迹。这个设计在单轮推理中尚可成立，但在 agent 与环境多次交互时会出现两个相互关联的失效模式。

第一，**采样预算没有对准真正的行为分岔点**。一条工具轨迹中，大量 token 只是沿既定思路继续展开，真正改变后续路径的往往是工具返回之后的少数决策：是否相信结果、是否改写查询、是否换工具、是否继续搜索。若所有预算都用于从题目起点重新生成完整轨迹，系统会反复支付相同前缀和相同工具调用的成本，却未必充分探索这些局部决策。

第二，**最终奖励无法直接说明哪一段交互值得保留**。传统做法把同一个轨迹级 advantage 广播给整条序列，因此只能回答“这条轨迹整体好不好”，不能区分共同前缀与分叉后缀各自对结果的贡献。尤其当多条候选路径共享同一段推理时，重复把各自的整轨迹 advantage 写回共享 token，会使训练信号的语义与采样树结构不一致。

作者的先导实验给出了一个可操作的切入点：模型在每次接收工具反馈后生成的前 10--50 个 token 上，entropy 明显上升；搜索返回后的上升又比确定性更强的 Python 返回更明显。这里的 entropy 不是正确性评分，而是策略分布的不确定性代理。论文据此判断，工具返回后的高熵区间更可能包含尚未充分探索的行为分支。

因此，ARPO 的贡献不是发明新的 reward，也不是更换一个全新的 RL objective，而是把**采样供给与 credit assignment 一起改造成树结构**：先在工具反馈后的高熵节点局部分叉，复用已经生成的共同前缀；再让共享前缀与独立后缀获得符合其归属关系的 advantage。其唯一承重叙事是：

> **用 entropy 找到值得追加探索的工具交互节点，用局部分叉把预算集中到这些节点，再用共享前缀感知的 advantage 将结果差异写回相应路径。**

这个定位也限定了“step-level”的含义。ARPO 没有学习 process reward model，也没有为每一步产生独立监督；它仍依赖最终答案奖励，只是借助局部分叉形成的反事实比较，把轨迹级信号更细地投射到共享段与分支段。因此更准确的说法是**树结构辅助的细粒度归因**，而不是严格意义上的逐步奖励。

## 机制与训练闭环

ARPO 的训练闭环由两个必须配套的改动组成：Adaptive Rollout 决定“在哪里多采”，Advantage Attribution 决定“采完以后如何把回报写回”。若只有前者，系统虽然生成了树状样本，却仍可能用不匹配的整轨迹方式更新共同前缀；若只有后者，又没有共享前缀下的局部分支可供比较。完整流程如下：

```text
冷启动 SFT policy + 问题 q + 工具环境 T
  -> 先生成 N 条完整轨迹，并记录各轨迹开头 k 个 token 的基准 entropy
  -> 每次工具返回后再观察 k 个 token，计算相对基准的 entropy 变化
  -> 若分支概率超过阈值，则从当前节点复制上下文并生成 Z 条局部后缀
  -> 在总预算 M 内形成“完整根轨迹 + 共享前缀 + 独立后缀”的 rollout group
  -> 用答案正确性、格式和多工具奖励得到每条终止路径的 outcome reward
  -> 对共享段聚合下游路径 advantage，对独立段保留各自 advantage
  -> 通过 GRPO 风格目标更新 policy
  -> 下一轮 rollout；部署时只保留最终 policy 与任务所需工具
```

### 环节一：用完整轨迹建立全局覆盖与 entropy 基线

给定每个问题的总 rollout 配额 $M$，ARPO 先从问题起点生成 $N$ 条完整轨迹，把剩余的 $M-N$ 个名额留给局部分叉。完整轨迹有两个作用：一是保留不同宏观解题路线，避免所有预算都困在某个局部前缀；二是为后续工具节点提供基准不确定性。对每条初始轨迹的前 $k$ 个 token，模型计算 token 分布 entropy：

$$
H_t=-\sum_{j=1}^{V}p_{t,j}\log p_{t,j},
\qquad
p_t=\pi_\theta(\cdot\mid R_{<t},x;\mathcal T)
=\operatorname{Softmax}(z_t/\tau).
$$

这些值组成 $H_{\text{initial}}\in\mathbb R^{1\times k}$。它描述的是当前 policy 在题目开头的生成不确定性，而不是一个跨问题统一的绝对阈值。用它作为同轨迹参照，意图是判断某次工具反馈是否使模型比初始状态更犹豫。

这里必须保留一部分完整采样。若 $N=0$，模型几乎只围绕少数局部前缀扩张，缺少全局路线覆盖；若 $N=M$，算法退化成普通完整轨迹采样。论文的 scaling experiment 在 $M=16$ 时发现 $N=8$ 最好，而 $N=16$ 明显下降，支持的不是某个普适最优比例，而是**全局路线与局部分支之间确实存在平衡**。

### 环节二：在工具反馈后的高熵节点按需分叉

每次工具返回 observation 后，policy 先生成 $k$ 个 token，并计算对应的 $H_t$。论文将其与初始基线作差后归一化：

$$
\Delta H_t=\operatorname{Normalize}(H_t-H_{\text{initial}}).
$$

随后将 entropy 变化映射为局部分支概率：

$$
P_t=\alpha+\beta\Delta H_t,
\qquad
\operatorname{Action}(P_t)=
\begin{cases}
\operatorname{Branch}(Z), & P_t>\tau,\\
\operatorname{Continue}, & \text{otherwise}.
\end{cases}
$$

$\alpha$ 提供基础分支倾向，$\beta$ 控制 entropy 变化对决策的影响，$\tau$ 是触发阈值，$Z$ 是一次复制出的后缀数。触发分叉时，系统不再从题目重新生成，而是复用当前节点之前的全部推理与工具历史，从该节点继续采样多个后缀。这样，同样的预算更直接地比较“面对相同工具反馈，接下来采取不同决策会怎样”。

局部分支总数达到 $M-N$ 后停止扩张，各路径继续生成到终止；若所有路径提前结束且局部预算没有用完，系统用新的完整轨迹补足总数。最终产物不是 $M$ 条彼此独立的序列，而是一组存在前缀共享关系的终止路径。

entropy 在这里仅承担**预算路由信号**。高 entropy 可能对应真正的策略转折，也可能只是搜索结果嘈杂、候选 token 同义或模型本身校准不佳。ARPO 没有证明高熵节点必然具有更高决策价值。超参实验中，entropy 权重从低值上升到 0.4 时性能提高、升到 1.0 时反而下降，恰好说明它只能作为适度偏置，不能替代全局多样性。

### 环节三：按共享前缀与独立后缀重新解释 advantage

局部分叉带来的关键结构是：若 $d$ 条终止路径从同一个节点分出，它们在分叉前共享 token，在分叉后才是各自的行为。ARPO 讨论了 hard 与 soft 两种归因方式。

Hard setting 先按组内最终奖励计算每条路径的标准化 advantage：

$$
\hat A_{i,t}=\frac{r_i-\operatorname{mean}(\{R_i\}_{i=1}^{G})}
{\operatorname{std}(\{R_i\}_{i=1}^{G})}.
$$

独立后缀中的 token 使用本路径 $\hat A_{i,t}$；被 $d$ 条路径共同拥有的前缀 token，则使用这些下游路径 advantage 的均值：

$$
\hat A^{\text{shared}}_t=\frac{1}{d}\sum_{i=1}^{d}\hat A_{i,t}.
$$

其因果含义是：共享前缀不是某一条后缀单独完成的动作，所以它的 credit 应反映从该前缀能够通向的整体结果；分叉后的动作才由各自结果单独评价。若同一前缀既导向成功也导向失败，其 advantage 会相互抵消，训练更倾向把差异归因到后缀，而不是错误地奖惩共同历史。

Soft setting 不显式改写 advantage，而是直接使用 GRPO 风格目标。共享前缀上的条件上下文完全相同，因此其 importance ratio 相同：

$$
r_{i,t}(\theta)=
\frac{\pi_\theta(y_{i,t}\mid x,y_{i,<t})}
{\pi_{\mathrm{ref}}(y_{i,t}\mid x,y_{i,<t})},
\qquad
y_{i,<t}=y_{j,<t}\Rightarrow r_{i,t}=r_{j,t}.
$$

在组损失求和时，共享 token 的同一 ratio 会同时乘上多条路径的 advantage，其梯度贡献因而近似于 hard setting 中的平均共享 advantage；分叉后上下文不同，ratio 也随路径分开。作者的训练曲线显示 soft setting 的 reward 更高且更稳定，因此最终默认使用 soft 方案。

这里的“soft attribution”主要是一种对既有 GRPO 目标在树状样本上的解释，而不是额外训练出的 estimator。它是否真的等价于理想的 stepwise credit，还受序列长度归一化、clipping、路径数量不均衡等因素影响。后续 AEPO 正是进一步处理树状 rollout 预算集中和高熵 token 被 clipping 的问题。

### 环节四：终局奖励闭合训练信号

每条分支最终仍通过 outcome reward 评价。论文沿用 Tool-Star 的分层奖励，把答案正确性、输出格式与多工具协同组合起来：格式正确且答案有部分或全部正确性时给予 accuracy reward，并在同时使用搜索与 Python 时加 $r_M=0.1$；格式正确但答案错误时为 0；格式错误时为 -1。

这部分不是 ARPO 的主要创新，却决定了树上每个叶节点最初拿到什么信号。它也引入一个需要单独看待的偏置：多工具 bonus 可能奖励确有必要的协同，也可能鼓励本可避免的额外调用。因此，实验反映的是“ARPO rollout/归因 + Tool-Star reward + 具体工具环境”这一完整 pipeline，不能把所有增益都归结为 entropy 机制。

论文还把局部分支段视为 macro-action，给出 generalized policy gradient：

$$
\nabla_\theta J(\theta)=
\mathbb E_{\tau\sim\pi_\theta}
\left[\sum_{T=1}^{K}
\nabla_\theta\log\pi_\theta(MA_T\mid MS_T)A_T(\tau)\right].
$$

这项分析说明，对可微 Transformer policy 而言，以连续 token 段作为宏动作进行 policy-gradient 更新在形式上成立，token-level policy gradient 是其特例。它提供的是**局部分支可以被合法优化**的理论基础，并不证明 entropy 是最优分支信号，也不直接推出 ARPO 比其他 tree-search 或 uncertainty routing 方法更优。

训练前，作者先用 Tool-Star 54K 与 STILL 0.8K 数据进行 cold-start SFT；深度推理 RL 使用 Tool-Star 10K 样本，deep-search RL 仅使用来自 SimpleDeepSearcher 与 WebSailor 的 1K 困难样本。搜索环境使用 Bing top-10 结果，数学与知识任务只使用 snippets，deep-search 还抓取网页并由 browser agent 精炼。Adaptive rollout、entropy 监测和 advantage attribution 都发生在训练阶段；推理时不需要保留训练用分支树，只部署更新后的 policy 和任务本身需要的工具。

## 证据、定位与边界

### 主结果：作为完整训练 pipeline，ARPO 稳定优于轨迹级基线

论文覆盖计算推理、知识推理和 deep search 共 13 个 benchmark。前两类任务的十项平均结果如下：

| Backbone | GRPO | DAPO | ARPO | 相对最佳基线 |
|---|---:|---:|---:|---:|
| Qwen2.5-3B | 50.4 | 50.6 | **52.8** | +2.2 |
| Llama3.1-8B | 51.1 | 50.4 | **55.3** | +4.2 |
| Qwen2.5-7B | 56.5 | 54.8 | **58.3** | +1.8 |

跨三个 backbone 的一致提升说明，这套训练 pipeline 不是只在单一模型上偶然奏效。它支持的核心结论是：对多轮工具任务，保留全局轨迹的同时在工具节点做局部探索，比完全独立的完整轨迹采样更有效。它不能单独证明 entropy routing、前缀归因和 reward design 中哪一项贡献最大，因为正文没有提供“去掉 entropy 分支”“保留分支但去掉 attribution”等完整组件移除消融。

在 deep-search 上，ARPO 同样优于 GRPO：

| Backbone | 方法 | GAIA | WebWalkerQA | HLE | XBench-DR |
|---|---|---:|---:|---:|---:|
| Qwen3-8B | GRPO | 32.0 | 29.0 | 7.8 | 20.0 |
|  | ARPO | **38.8** | **30.5** | **8.8** | **25.0** |
| Qwen3-14B | GRPO | 36.9 | 30.0 | 8.6 | 27.0 |
|  | ARPO | **43.7** | **36.0** | **10.0** | **32.0** |

Pass@5 下，Qwen3-14B + ARPO 在 GAIA、HLE、XBench-DR 分别达到 61.2、24.0、59.0，说明训练后策略仍具有可利用的采样多样性。不过 XBench-DR 的 Pass@K 实验改用了中文 prompt，和主表并非完全同设置，不能直接把差值都解释为 ARPO 的 scaling 收益。

### 机制证据：支持“需要平衡”，尚未完成严格因果拆分

Qwen2.5-7B 的超参分析呈现出三条有意义的趋势。第一，entropy 影响适中时性能最好，过强时下降，说明高熵路由是偏置而不是越多越好。第二，在 $M=16$ 下，初始完整采样数从 0 增至 8 时改善，增至 16 时下降，说明纯局部分叉和纯全局采样都不是最佳配置。第三，更大的总 rollout size 带来更高性能，说明方法能够从更多采样预算中获益。

这些结果验证了 ARPO 的设计张力，却还不是严格消融。论文没有报告等成本的随机节点分叉、基于 reward/value 的分叉、只做前缀复用、不做 advantage attribution 等对照，因此无法判断 entropy 相对其他不确定性或价值信号的独特优势。Hard 与 soft 的训练 reward 曲线只说明 soft 版本在该设置中更稳定，也不足以证明其归因更接近真实因果贡献。

### 效率证据：工具调用减少是真实卖点，但口径仍需收紧

论文在 Qwen2.5-7B 上比较训练过程，ARPO 以约一半工具调用次数取得高于 GRPO 的总体准确率。这与前缀复用和只在高熵节点追加分支的机制方向一致：多个后缀不必重复支付从题目到分叉点的工具调用。

但该结论来自单一 backbone 的图示，而非跨模型、跨任务的统一成本表。论文给出的 rollout 复杂度从常规 $O(n^2)$ 降至 $O(n\log n)$ 与 $O(n^2)$ 之间，也明确忽略 token entropy 计算开销。真实系统成本还取决于 KV-cache 复用、分支调度、网页请求延迟和 browser agent 成本。因此，“一半工具预算”应理解为论文特定训练配置下的观测，不宜外推为普遍的 wall-clock 或总算力减半。

### 外部工具能力是重要混杂因素

Browser ablation 显示，Qwen3-14B 在 GAIA、HLE、WebWalkerQA 上仅用 snippets 的平均分为 24.8，换成同规模 browser agent 后为 29.9，换成 QwQ-32B browser 后达到 39.4。外部 browser 的能力本身足以造成大幅变化。这说明 deep-search 结果不仅由 policy optimization 决定，也由检索覆盖、页面抓取和信息精炼模型共同决定。ARPO 证明的是在其固定工具栈内更有效，而不是脱离工具质量仍能稳定泛化。

### 方法定位：ARPO 定义了 agentic RL 的“树状采样 + 树感知归因”路线

与 Tool-Star、OTC 等主要从训练框架或 reward 约束工具行为的方法相比，ARPO 把优化重心移到 rollout topology：不是只评价工具是否使用得对，而是改变候选交互行为怎样被生成和比较。与 process reward 方法相比，它不新增逐步 verifier，而是继续使用 outcome reward，通过共享前缀下的分支结果形成更细的比较。与普通 tree search 相比，它的目的也不是推理时搜索最优答案，而是训练时制造更有信息量、成本更低的 policy-gradient 样本。

它更像一个基础范式，而不是已经封闭的最终算法。后续 AEPO 指出了两类残余问题：高熵节点连续触发可能使分支预算集中在少数根轨迹，标准 clipping 又可能抹掉高熵 token 的有效梯度。因而 ARPO 最重要的价值，是明确了**多轮 agent RL 必须同时设计采样树与更新语义**；其具体 entropy 路由和 soft attribution 仍有稳定性与因果性空间。

| 前提或代价 | 为什么重要 | 可能的失败模式 |
|---|---|---|
| entropy 与高价值行为分岔相关 | 决定局部分支落点 | 噪声工具结果造成高熵但无价值的扩张 |
| outcome reward 能区分路径质量 | 所有 advantage 最终都来自叶节点奖励 | reward hacking 或稀疏奖励污染整棵树的归因 |
| 共享前缀可被高效复用 | 决定工具和生成成本优势 | 系统不支持 KV/cache 与环境状态复用时收益缩水 |
| 分支预算不过度集中 | 决定全局覆盖 | 连续高熵使少数路径耗尽预算 |
| 外部工具稳定且可复现 | 决定训练信号与 benchmark 可比性 | 搜索索引、网页内容或 browser agent 变化导致结果漂移 |
| soft attribution 的优化近似成立 | 决定共享 token 的更新语义 | clipping、长度归一化和不均衡分支破坏平均归因 |

## 研究卡片

**问题。** 完整轨迹独立采样把预算浪费在重复前缀上，轨迹级 advantage 又无法区分共享推理与局部工具决策，使多轮 agent-environment interaction 的关键分岔既未被充分探索，也未被恰当归因。

**做法。** 保留 $N$ 条完整轨迹维持全局覆盖，把剩余 $M-N$ 个名额用于工具反馈后的高熵节点分叉；由此形成共享前缀与独立后缀组成的 rollout tree，再以 hard 显式平均或 soft GRPO 聚合的方式，为共享段汇总下游 advantage、为独立段保留本路径 advantage。

**证据。** ARPO 在 Qwen2.5-3B、Llama3.1-8B、Qwen2.5-7B 的十项推理平均分上均优于 GRPO/DAPO，在 Qwen3-8B/14B 的四项 deep-search benchmark 上均优于 GRPO；Qwen2.5-7B 实验中以约一半工具调用取得更高准确率。超参趋势支持全局采样与局部分叉需要平衡，但缺少完整组件移除和等成本替代路由对照。

**前提。** 工具返回后的 entropy 上升必须与值得探索的行为分岔有足够相关性；叶节点 outcome reward 必须可靠；训练系统需要复用共享上下文和工具状态，否则局部分叉的成本优势难以兑现。

**教训。** 多轮 agent RL 的关键不只是选择 GRPO、DAPO 或某个 reward 公式，而是让**样本如何生成、样本之间有什么结构、回报如何沿该结构写回**三者一致。所谓 step-level 学习不一定要求额外的 process reward，也可以来自同一前缀下受控分支所形成的结果比较；但应准确区分“更细的归因”与“真正的逐步监督”。

**遗留问题。** entropy 是否优于 value disagreement、reward uncertainty 或语义分歧作为分叉信号？如何在不同根轨迹间公平分配分支预算？共享前缀的 soft advantage 在 clipping 和长度归一化下是否仍保持期望语义？若计入网页请求、browser agent、entropy 计算和调度开销，ARPO 的端到端成本优势还剩多少？这些问题决定它能否从有效 pipeline 进一步变成稳定、可扩展的通用 agentic RL 算法。
