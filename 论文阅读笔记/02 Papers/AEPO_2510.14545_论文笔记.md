# AEPO（arXiv:2510.14545）论文笔记

Guanting Dong et al. | WWW 2026 稿件版式 / arXiv v1, 2025 | [PDF](https://arxiv.org/pdf/2510.14545v1) | `Agentic RL` `Entropy-Balanced Rollout` `Gradient-Preserving Clipping`

## 问题与核心判断

>ARPO 的核心做法
>
>1. **对 agent 的决策进行 rollout（ rollout = 让 agent 实际跑一遍，生成完整轨迹）**
>2. **在 rollout 过程中，如果在某个步骤 agent 很"不确定"（entropy 高），就分叉（branch/fork）**，产生多条不同的后续轨迹
>3. **用这些轨迹的数据去更新 policy（策略/模型参数）**

AEPO 关注的不是 web agent 有没有探索，而是**entropy-guided exploration 能否被均匀采到，并真正学进参数**。ARPO 一类 agentic RL 方法会在高熵工具调用步骤处分叉，以探索同一前缀下不同的后续工具行为；但把“高 entropy”直接当作分支依据，会在 rollout 与 policy update 两个阶段分别产生一种自相矛盾的失效。



| 指标 | 计算对象 | 公式本质 | 值越大表示… |
|:--|:--|:--|:--|
| **Entropy $H_t$** | 概率分布的平坦程度 | $-\sum p_i\log p_i$ | 模型在多个选项间摇摆 |
| **PPL (Perplexity)** | 模型对接下来 token 的"吃惊程度" | $\exp(-\frac{1}{N}\sum\log p(\text{token}_i))$ | 模型对整体序列没把握 |

**Entropy** 关心单步分布的形状（犹豫不犹豫），不关心实际选了哪个 token；**PPL** 关心整条序列上模型猜没猜中实际 token。两者相关但不相同——高熵步也可能 PPL 很低（如果真实 token 恰好落在平坦分布里的高概率点上）。

> **大白话**：某些工具调用结果会让 agent 连续多个步骤都处于"犹豫/不确定"状态（高熵），每次犹豫都触发一次分叉，于是一条路径上不断分叉再分叉，把预算提前花光。其他路径即使也可能有正确解法，却没钱（预算）展开了。数据显示连续高熵占 56.5%，比孤立高熵的 43.5% 更多；预算只有 8 个分支时，93.4% 的分支集中在 1-3 条路径上。所谓的 collapse 不是不做探索，而是**探索树长歪了——只往几条路猛挖，其他路根本没挖**。

第一种失效是 **High-Entropy Rollout Collapse**。工具结果可能连续触发多个高熵推理轮次，于是同一条路径会反复满足分支条件，提前耗尽局部 branch budget；其他根轨迹即使也包含潜在正确分支，却没有预算展开。论文在 ARPO 的 1K 样本训练设置中发现，连续高熵工具轮次占 `56.5%`，高于孤立高熵轮次的 `43.5%`；当分支预算为 8 时，`93.4%` 的分支集中在 1-3 条轨迹上。这里所谓 collapse 不是完全没有探索，而是**搜索树在少数路径上局部过度生长**。

> **大白话**：高熵 token 往往出现在逻辑转折、反思、选工具这类关键位置，而它们的 importance ratio（新旧 policy 的概率比值）也更容易超过 clipping 上界。一旦超界，标准 clipping 直接把这些 token 的梯度设为 0。这就矛盾了：rollout 阶段花了工具调用成本辛苦探索出来的行为，到了 update 阶段反而被"砍掉"，学不进参数。即使 DAPO 放宽了 clipping 上界，大量高熵 token 仍然被裁掉。

>**高熵 = 多个 token 的概率差不多，分布平坦，模型"犹豫"**  
   **低熵 = 某个 token 概率明显大于其他，分布尖锐，模型"确定"**

第二种失效是 **High-Entropy Token Gradient Clipping**。高熵 token 往往对应逻辑转折、反思或工具选择等改变行为路径的位置，其 importance ratio 也更容易越过 PPO/GRPO clipping 上界。标准 clipping 会让正优势、上界越界 token 的梯度变为 0。结果是：rollout 阶段通过树形分支付出额外工具成本找到的探索行为，在 update 阶段反而可能没有梯度进入 policy。即使 DAPO 放宽上界，作者观察到大量高熵 token 仍被裁掉。

因此，AEPO 不是一篇简单的“提高 policy entropy”论文，也不是只换一个 branch probability。它真正修补的是 entropy 信号在两阶段之间的断裂：rollout 端要避免高熵让预算过度集中，update 端又要避免 clipping 把成功探索的梯度抹掉。单独修前者，只能得到更多样的轨迹但未必学得进去；单独修后者，则没有保证被保留梯度的候选来自充分覆盖的探索空间。

所以，**先按根/工具熵差分配全局与局部预算，再用连续分支惩罚扩大轨迹覆盖，最后用 entropy-aware advantage 与 stop-gradient clipping 把成功探索写回 policy**，是整篇论文唯一承重的叙事。

## 机制与训练闭环

AEPO 把同一类高熵信号贯穿两个阶段，但角色不同：在 rollout 阶段，entropy 是资源调度和分支路由信号；在 update 阶段，entropy 是 token-level advantage shaping 信号。两者之间的中间产物是更分散的 rollout group，以及其中被 outcome reward 证明有价值的高不确定 token。

```text
问题 q + 工具环境 T
  -> 预探索 1 条完整轨迹，估计根 entropy 与工具后平均 entropy
  -> 将总预算 k 动态拆成 m 条全局轨迹 + (k-m) 条局部分支
  -> 对每条轨迹监测 entropy variation，并惩罚连续高熵分支
  -> 得到覆盖更分散的 rollout group P + trajectory reward
  -> group-relative outcome advantage + token entropy advantage
  -> stop-gradient clipping 保留正优势越界 token 的有界梯度
  -> 更新 policy；推理时只部署最终模型和任务所需工具
```

---

### 环节一：Entropy Pre-Monitoring——先决定预算从哪里开始花

**核心思想**：固定“完整轨迹数/局部分支数”默认所有问题的不确定性结构相同，但有的问题从题面开始就存在多条宏观路线，有的问题则是在某次工具 observation 返回后才产生分歧。AEPO 先判断不确定性主要来自根节点还是工具节点，再分配 rollout budget。

对每个问题，系统额外生成 1 条完整工具轨迹作为预监测样本，估计问题根部的 entropy $H_{\mathrm{root}}$，以及各工具调用后的平均 entropy $H_{\mathrm{tool}}^{\mathrm{avg}}$。给定总预算 $k$，完整全局轨迹数量为：

$$
m=k\cdot\sigma\!\left(\beta\left(H_{\mathrm{root}}-H_{\mathrm{tool}}^{\mathrm{avg}}\right)\right).
$$

| 熵差 | 预算变化 | 直觉 |
|:--|:--|:--|
| $H_{\mathrm{root}}>H_{\mathrm{tool}}^{\mathrm{avg}}$ | 增加 $m$，多从根节点重启 | 问题本身路线不确定，不应过早锁定一条主链 |
| $H_{\mathrm{root}}<H_{\mathrm{tool}}^{\mathrm{avg}}$ | 减少 $m$，增加 $k-m$ 局部分支 | 分歧主要由工具结果引入，应把预算留给中间节点 |

这一公式把固定采样比例改成按问题自适应的资源调度。它借用了“信息增益与 token entropy 正相关”的启发式，但并非严格求解信息增益最大化；实际系统还需对 $m$ 做离散化。代价是每个问题额外执行一条完整预探索轨迹，因此它优化的是后续预算分布，不是消除采样成本。

> **预探索的本质是一次 rollout profiler。** 它不直接提供训练样本或 reward，而是输出本题的预算配置 $(m,k-m)$，供下一环节执行。

>AEPO 先花一条轨迹的钱测一下，**你这道问题的不确定性到底在开头还是在中间**，然后再决定怎么花剩下的预算。
---

### 环节二：Entropy-Balanced Adaptive Rollout——让高熵影响分支，但不能独占分支

**核心思想**：entropy variation 决定“当前节点值不值得展开”，连续分支惩罚决定“这条链是否已经吃掉过多预算”。AEPO 不是减少探索量，而是重新分配探索的空间覆盖。

系统先生成 $m$ 条全局轨迹。每次工具调用后，监控当前输出相对初始状态的标准化 entropy change $Delta H_t$。在第 $t$ 个工具步骤，分支概率为：

$$
P_t=(\alpha+\gamma\Delta H_t)\left(1-\widehat{P}(l)\right).
$$

$\alpha$ 提供基础分支率，$\Delta H_t$ 让工具结果引起的不确定性上升提高分支倾向；$l$ 记录连续高熵分支状态，$\widehat{P}(l)$ 随 $l$ 增长，对同一路径的连续展开施加惩罚。若 $P_t>\tau$，当前节点生成 $Z$ 条局部后缀；否则继续原轨迹：

$$
\mathrm{Action}(P_t)=
\begin{cases}
\mathrm{Branch}(Z), & P_t>\tau,\\
\mathrm{Continue}, & \text{otherwise}.
\end{cases}
$$

分支数达到局部预算 $k-m$ 后停止继续分叉；若所有轨迹提前结束、局部预算仍未用完，系统以额外完整轨迹补齐。最终输出的是 rollout pool $P$，它同时包含从根节点独立生成的 global trajectories 和从工具节点展开的 partial trajectories。

| | ARPO | AEPO |
|:--|:--|:--|
| 分叉条件 | 只要 entropy 高就分叉 | entropy 高 + 没吃太多预算 |
| 对连续分叉 | 无限制，一条链不断分叉 | 有惩罚，压住局部过度生长 |
| 预算用不完时 | 直接结束 | 用额外完整轨迹补齐，不浪费 |

```text
ARPO：一条连续高熵链 -> 分支 -> 再分支 -> 提前耗尽预算

AEPO：根轨迹 1 -> 有限分支
      根轨迹 2 -> 有限分支
      根轨迹 3 -> 有限分支
      余额      -> 补充完整轨迹
```

这个设计解决的是候选分布问题，而不是直接提高 reward。它期望更多根轨迹有机会贡献局部分支，从而让 group-relative update 看到更丰富的工具行为。代价是树形 rollout、entropy 在线监控和外部工具执行共同增加系统复杂度。

>AEPO 要管的是**探索树不要往一条路猛挖**，所以它限制的是**分叉的次数/轨迹的条数**，不是每条轨迹写多长。
---

### 环节三：Entropy-Aware Advantage——指出哪些探索 token 更值得学

**核心思想**：trajectory reward 只能说明整条轨迹是否成功；若同一个 outcome advantage 平铺给所有 token，模型无法区分低风险连接词与真正改变工具行为的高不确定决策。AEPO 因此用 token entropy 调整组内 outcome advantage。

先计算 group-relative accuracy advantage 与 token entropy z-score：

$$
\widetilde{A}_{\mathrm{Acc}}^{(t)}=
\frac{r_t-\operatorname{mean}(\{R_i\}_{i=1}^{G})}
{\operatorname{std}(\{R_i\}_{i=1}^{G})},
\qquad
\widetilde{A}_{\Delta H}^{(t)}=
\frac{H_t-\operatorname{mean}(\{H_t\}_{t=1}^{T})}
{\operatorname{std}(\{H_t\}_{t=1}^{T})}.
$$

再将两者相乘：

$$
\widetilde{A}^{(t)}=
\widetilde{A}_{\mathrm{Acc}}^{(t)}
\left(1+a\widetilde{A}_{\Delta H}^{(t)}\right).
$$

对于成功轨迹，entropy 高于轨迹平均值的 token 获得更大的正优势，因此逻辑转折、反思与工具选择等高不确定决策被优先学习；对于失败轨迹，entropy 也会调制负优势。entropy 在这里不是独立 reward，而是 **credit reweighting factor**：它不决定轨迹答对与否，只改变同一 outcome 下不同 token 的学习强度。

> **这一环没有得到真正的 step-level verifier。** $H_t$ 只能表示 policy 不确定，不等于该 token 对最终成功有因果贡献；AEPO 只是用 entropy 对 trajectory-level credit 做细粒度重加权。

---

### 环节四：Stop-Gradient Clipping——让成功探索真正进入参数 ⭐ 核心创新

**核心思想**：advantage 已经指出高不确定 token 值得学，但如果其 importance ratio 越界后梯度仍为 0，前面的 rollout 和 credit shaping 都无法转化为 policy 更新。

token importance ratio 为：

$$
\delta_t(\theta)=
\frac{\pi_\theta(a_t\mid s_t)}
{\pi_{\theta_{\mathrm{old}}}(a_t\mid s_t)}.
$$

标准 clipping 对正优势且 $\delta_t>1+\epsilon_h$ 的 token 截断梯度。AEPO 参考 GPPO，在 clipping 上界中加入 stop-gradient，使前向值保持受限，反向却保留一个有界梯度。其梯度缩放可以写成：

$$
F_{j,t}(\theta)=
\begin{cases}
1+\epsilon_h, & \widetilde{A}^{(t)}>0\ \text{and}\ \delta_t>1+\epsilon_h,\\
0, & \widetilde{A}^{(t)}<0\ \text{and}\ \delta_t<1-\epsilon_l,\\
\delta_t, & \text{otherwise}.
\end{cases}
$$

| 区域 | 标准 clipping | AEPO |
|:--|:--|:--|
| 正优势、ratio 超上界 | 梯度归零 | 以 $1+\epsilon_h$ 保留有界梯度 |
| 负优势、ratio 低于下界 | 梯度归零 | 仍然归零 |
| clipping 区间内 | 正常更新 | 正常更新 |

AEPO 因而不是取消 clipping，也不是对所有高 entropy token 无条件放大，而是只在“当前 advantage 为正且 policy ratio 超上界”时保留更新。需要特别注意：公式里的直接 gate 是 advantage 与 ratio，不是 entropy mask；“这些 token 往往高熵”的解释依赖作者观察到的 entropy-ratio 相关性。真正显式使用 $H_t$ 的是上一环的 advantage shaping。

训练时工具返回文本被排除在 loss 之外，避免把 environment observation 当作 policy action 学习。参数更新完成后，预监测、树形分支、entropy advantage 和 stop-gradient 都是训练期机制；推理时只部署最终 policy，并保留任务本身所需的搜索、浏览和代码工具。

## 证据、定位与边界

### 主实验结果

AEPO 最可信的结果是：在其直接前身 ARPO 上，deep-search 与通用 reasoning 两类设置都取得一致但幅度有限的提升。

| Qwen3-14B，Pass@1 | ARPO | AEPO | 绝对提升 |
|:--|--:|--:|--:|
| GAIA | 43.7 | **47.6** | +3.9 |
| WebWalkerQA | 40.5 | **44.5** | +4.0 |
| HLE | 10.0 | **11.2** | +1.2 |
| XBench-DR | 32.0 | **35.0** | +3.0 |
| FRAMES | 55.4 | **58.8** | +3.4 |

Pass@5 的提升也持续存在：GAIA 从 `61.2` 升至 `65.0`，HLE 从 `24.0` 升至 `26.0`，WebWalkerQA 从 `62.0` 升至 `70.0`。在另外 10 个数学与知识密集任务上，Llama3.1-8B 的平均分从 ARPO 的 `55.3` 升到 `56.3`，Qwen2.5-7B 从 `58.3` 升到 `60.1`。这说明收益不只依赖单一 backbone，但不能据此把提升描述成数量级变化。

原文摘要称 Qwen3-14B 的 WebWalkerQA Pass@1 为 `43.0`，Table 1 与 Figure 5 则给出 `44.5`；笔记采用表格值，并保留这一版本内部不一致。

### 机制证据：方向一致，但不能替代消融

| 观察 | ARPO | AEPO | 能支持什么 |
|:--|:--|:--|:--|
| rollout embedding 聚类中心 | 54 | 62 | 采样表征覆盖更分散 |
| 分支覆盖 | 通常集中于 2-3 条轨迹 | 最多覆盖全部 8 条路径 | branch budget 分布更均衡 |
| 工具调用数 | 高于 AEPO | 约为 vanilla/clipping 方法的一半，且低于 ARPO | 训练期工具消耗更低 |
| policy entropy 曲线 | 更易波动 | 更高且更平稳 | update dynamics 与设计目标一致 |

这些结果与“扩大 rollout 覆盖、稳定 entropy 更新”的叙事一致，但强度有限。BGE-M3 embedding + PCA + DBSCAN 衡量的是表征空间聚类，不保证轨迹在功能上真正不同；工具调用少也不自动等于真实成本低；entropy 曲线稳定更是相关性证据，不能单独证明 stop-gradient 或 advantage shaping 是原因。

### 最关键的证据缺口：没有逐组件消融

论文没有分别去掉以下组件：

- entropy pre-monitoring
- global/branch 动态预算分配
- consecutive branch penalty
- entropy-aware advantage
- stop-gradient clipping

因此，实验能证明的是**整体 AEPO pipeline 优于 ARPO 与其他 baselines**，不能证明五个组件都必要，也不能判断收益主要来自 rollout 分布、advantage shaping，还是 gradient preservation。与 EnvScaler、SmartSearch、ET-Agent 的笔记标准一致，这里必须把 benchmark improvement 与 mechanism proof 分开。

### 方法定位

AEPO = **`ARPO-style entropy-guided tree rollout + adaptive budget balancing + entropy-shaped token advantage + GPPO-style gradient-preserving clipping`**

它不是从头定义新的 agentic RL 家族，而是 ARPO 的 stabilizer：ARPO 提出在高熵工具步骤做树形分支，AEPO 进一步指出“越依赖 entropy，越可能出现局部分支集中和高熵梯度截断”，于是同时修 rollout budget 与 policy update。

与 DAPO/GPPO/CISPO 等 clipping-optimized 方法相比，AEPO 的区别也不只在 clip 公式。它先用 agent interaction 中的 entropy 组织候选轨迹，再将同一信号用于 token advantage；gradient-preserving clipping 是闭环最后一环，而非孤立技巧。

### 代价与前提

| 局限 | 原因 |
|:--|:--|
| entropy 是代理变量 | 高 entropy 可能是有效决策分歧，也可能是句法变体、API 不稳定或模型无知 |
| 无逐组件消融 | 只能验证整套 pipeline，无法精确归因哪个设计承重 |
| clipping 并非显式 entropy gate | 梯度保留由正优势与 ratio 越界触发，高熵解释依赖经验相关性 |
| 公式与伪代码记号不一致 | 正文用 $\gamma$ 缩放 $\Delta H_t$，Algorithm 1 用 $\beta$；连续惩罚参数的角色也有互换 |
| stop-gradient 文字存在代数错误 | 正文声称 $\delta\cdot\operatorname{sg}(\delta)=1$，按实际结构应是 $\delta/\operatorname{sg}(\delta)=1$；复现应以代码为准 |
| rollout 成本高 | 每题额外预探索，随后执行全局轨迹、局部分支和外部搜索/浏览/代码工具 |
| 评测依赖外部系统 | 使用 Bing Search API，多数任务由 Qwen2.5-72B-Instruct judge；索引时效性和 judge 偏差影响复现 |
| 算力与统计报告有限 | 训练使用 16 张 H800、20K context，未报告多随机种子与显著性 |

## 研究卡片

- **问题**：entropy-guided agentic rollout 会把分支预算耗在少数连续高熵轨迹上，而 policy clipping 又会截断这些探索 token 的梯度，造成“采到了但学不进去”。
- **做法**：预探索估计根/工具 entropy -> 动态分配全局与局部预算 -> 连续分支惩罚扩大根轨迹覆盖 -> entropy-aware advantage 标出高不确定 token -> stop-gradient clipping 保留成功越界 token 的有界梯度。
- **证据**：Qwen3-14B 在 5 个 deep-search benchmark 上均超过 ARPO，WebWalkerQA Pass@5 从 `62.0` 提升到 `70.0`；分支覆盖、聚类数和 entropy curve 与动机一致，但没有逐组件消融。
- **前提**：token entropy 与有价值的行为不确定性足够相关，importance ratio 越界的正优势 token 确实包含值得保留的探索行为，训练系统也能承担树形 rollout 和外部工具成本。
- **教训**：探索算法不能只优化“采到什么”，还必须追踪这些候选在 update 阶段是否获得有效梯度。对长程 Agent RL，rollout budget、credit shaping 和 clipping 应围绕同一个失效模式联合设计。
- **遗留**：需要严格消融预算调度、连续惩罚、entropy advantage 与 gradient preservation 的各自贡献；还应比较 verifier disagreement、value uncertainty 或语义级不确定性是否比原始 token entropy 更可靠。
