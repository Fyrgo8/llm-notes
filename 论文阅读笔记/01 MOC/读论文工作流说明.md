# 读论文工作流说明

## 目标

这套工作流不是只把 Zotero 文献“导进来”，而是把一篇论文从：

- 文献资产
- PDF 批注
- intake note
- deep-read final note
- topic knowledge base

串成一条稳定链路。

## 两层笔记模型

### 第一层：intake/import note

模板：
[[07 Templates/Zotero Literature Note Import|Zotero Literature Note Import]]

作用：

- 承接 Zotero 的 canonical metadata
- 承接 bibliography / abstract / PDF link
- 承接 annotations / image exports

它更像“入库笔记”，不是最终研究笔记。

### 第二层：deep-read final note

模板：
[[07 Templates/Deep Read Final Note Template|Deep Read Final Note Template]]

作用：

- 输出精读后的最终成稿
- 风格对齐 `02 Papers/ET-Agent_2601.06860_论文笔记.md`
- 保留机制拆解、公式解释、实验判断、风险分析、最终 notes block

它才是以后真正反复回看的笔记。

## 推荐流程

1. 先在 Zotero 导入论文元信息和 PDF
2. 在 Zotero 里做高亮、批注、区域截图
3. 用 Obsidian Zotero Integration 导入，得到 intake note
4. 如果这篇论文值得精读，再基于原文 + intake note 产出 final note
5. final note 放进 `02 Papers/`
6. 再把它挂到 `03 Topics/` 对应主题页

## paper-deepread skill 在这套流程里的位置

`paper-deepread` 不应该只做“单次解释”，而应该负责：

- 读取论文原文
- 做 triage / deepread / notes extraction
- 最终产出一篇可以直接进入 `02 Papers/` 的 deep-read final note

默认目标产物：

- 风格接近 `ET-Agent_2601.06860_论文笔记.md`
- 比 intake note 更重机制解释
- 比 Zotero raw import 更强调研究判断

## 文件夹映射

- `02 Papers/`：最终论文笔记
- `03 Topics/`：主题知识页
- `08 Attachments/`：导出的 annotation 图片
- `09 Excerpts/`：原始摘录、批注块、过渡材料

## 最终原则

- Zotero 管 canonical 文献元信息和 PDF 批注
- Obsidian 管深读成稿、主题组织和跨论文链接
- `paper-deepread` 负责把“读懂”真正变成“可沉淀资产”
